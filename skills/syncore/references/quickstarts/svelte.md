# Svelte Quickstart

Use this setup when the app host is Svelte or SvelteKit and the runtime should
still stay fully local in a browser worker.

## 1. Create the app host

Start from a Svelte or SvelteKit app that already builds.

## 2. Install packages

```bash
npm install syncorejs svelte
```

## 3. Scaffold Syncore

```bash
npx syncorejs init --template svelte
```

This creates `syncore/schema.ts`, `syncore/functions/`, `syncore.config.ts`,
`src/syncore.worker.ts`, and `src/SyncoreProvider.svelte`. The auto-detection
in `syncorejs init` also recognises Svelte projects from `package.json`
dependencies (`svelte`, `@sveltejs/vite-plugin-svelte`, `@sveltejs/kit`).

## 4. Start the Syncore dev loop

```bash
npx syncorejs dev
```

For Svelte apps, operational commands run against connected `client:<id>` targets.

## 5. Confirm the scaffolded worker

`src/syncore.worker.ts` (generated by `init`):

```ts
/// <reference lib="webworker" />

import { createBrowserWorkerRuntime } from "syncorejs/browser";
import schema from "../syncore/_generated/schema";
import { functions } from "../syncore/_generated/functions";

void createBrowserWorkerRuntime({
  endpoint: self,
  schema,
  functions
});
```

For production apps, pass `resolvedComponents`, `databaseName`,
`persistenceDatabaseName`, and `persistenceMode` as needed:

```ts
import { resolvedComponents } from "../syncore/_generated/components";

void createBrowserWorkerRuntime({
  endpoint: self,
  schema,
  functions,
  components: resolvedComponents,
  databaseName: "my-syncore-svelte",
  persistenceDatabaseName: "my-syncore-svelte",
  persistenceMode: "opfs",
  platform: "browser-worker"
});
```

## 6. Use the scaffolded provider

`src/SyncoreProvider.svelte` (generated by `init`) is a Svelte 5 component that
creates the browser worker client, registers it globally with `setSyncoreClient`,
and disposes it when destroyed:

```svelte
<script lang="ts">
  import { onDestroy } from "svelte";
  import type { Snippet } from "svelte";
  import { createBrowserWorkerClient } from "syncorejs/browser";
  import { setSyncoreClient } from "syncorejs/svelte";

  interface Props {
    children?: Snippet;
  }

  const { children }: Props = $props();

  const managed = createBrowserWorkerClient({
    workerUrl: new URL("./syncore.worker.ts", import.meta.url)
  });

  setSyncoreClient(managed.client);

  onDestroy(() => {
    void managed.dispose();
  });
</script>

{@render children?.()}
```

Mount it at the root of your app so all descendant components can use Syncore
stores:

`src/App.svelte`

```svelte
<script lang="ts">
  import SyncoreProvider from "./SyncoreProvider.svelte";
  import Tasks from "./Tasks.svelte";
</script>

<SyncoreProvider>
  <Tasks />
</SyncoreProvider>
```

## 7. Query and mutate from Svelte components

`src/Tasks.svelte`

```svelte
<script lang="ts">
  import {
    createMutation,
    createQueryStore,
    createSyncoreStatusStore
  } from "syncorejs/svelte";
  import { api } from "../syncore/_generated/api";

  const tasksStore = createQueryStore(api.tasks.list);
  const runtimeStore = createSyncoreStatusStore();
  const createTask = createMutation(api.tasks.create);
</script>

{#if $runtimeStore.kind !== "ready"}
  <p>Syncore status: {$runtimeStore.kind}</p>
{:else}
  <button onclick={() => void createTask({ text: "Track from Svelte" })}>
    Add task
  </button>
  <pre>{JSON.stringify($tasksStore.data ?? [], null, 2)}</pre>
{/if}
```

## 8. Reach for richer stores when the app needs them

- use `createQueryStore(...)` for the simple path
- use `createQueriesStore(...)` for keyed compositions
- use `createPaginatedQueryStore(...)` for app-ready pagination
- use `createSyncoreStatusStore()` for local runtime lifecycle

## 9. Run the app

```bash
npm run dev
```
